# Smart Parking: рекомендуемый сценарий внешнего demo

Дата: 2026-04-21
Источник: выводы из `docs/FRONTEND_FINAL_AUDIT_2026-04-20.md`.

## 1) Какие маршруты показывать в demo

### Базовые demo-экраны (рекомендуется)
1. `/dashboard` — общий вход и ключевые KPI.
2. `/analytics` — аналитика, прогноз, аномалии.
3. `/bookings/new` — сценарий создания бронирования.
4. `/my-bookings` — пользовательский контроль своих заявок.
5. `/booking-management` — операционные действия (для ролей admin/owner/guard).
6. `/parking-lots` и `/parking-spots` — подтверждение «реального» справочника, без имитации.

### Дополнительно для admin-demo
1. `/admin` — единая точка входа в админ-функции.
2. Переходы с `/admin` в `/admin-users` и `/audit-logs` по сценарию «контроль доступа и проверка действий».

## 2) Что скрыто из основного navigation UX

Чтобы убрать ощущение transitional UI и сделать маршрут показа линейным:
- `/admin-users` убран из бокового меню (доступен через `/admin`).
- `/audit-logs` убран из бокового меню (доступен через `/admin`).
- `/home` и `/bookings` оставлены как служебные маршруты обратной совместимости, но не участвуют в demo-навигации.

## 3) Рекомендуемый маршрут показа (8–12 минут)

### Вариант для роли admin/owner
1. **Обзор системы** (`/dashboard`) — показать состояние сервиса и базовые метрики.
2. **Аналитика и прогноз** (`/analytics`) — продемонстрировать data-driven блоки.
3. **Создать бронирование** (`/bookings/new`) — пройти пользовательский CTA «от выбора к подтверждению».
4. **Мои бронирования** (`/my-bookings`) — показать прозрачность статусов после создания.
5. **Операции по бронированиям** (`/booking-management`) — подтвердить управляемость жизненного цикла.
6. **Справочники** (`/parking-lots` или `/parking-spots`) — показать, что сценарий опирается на реальные сущности.
7. **Администрирование** (`/admin`) — при необходимости завершить блоком про роли и аудит.

### Вариант для tenant
1. `/my-bookings`
2. `/bookings/new`
3. `/analytics`
4. `/dashboard`

## 4) Контроль перед показом

- Интерфейс и CTA должны быть русскоязычными и понятными без внутреннего контекста.
- Не заходить в служебные маршруты (`/home`, `/bookings`) в публичном показе.
- Для админ-блока входить через `/admin`, а не через технически «плоскую» навигацию.
