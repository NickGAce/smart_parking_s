# Smart Parking — glossary и стайлгайд пользовательского copy

Дата: 2026-04-21

## Принципы
- Пишем коротко и по делу.
- Используем русский язык в пользовательском интерфейсе.
- Технические коды (`no_show`, `old_values`, `assignment_metadata`) не показываем пользователю в сыром виде.
- Избегаем смешения RU/EN в одной фразе.

## Единые термины

### Бронирование
- Основной термин: **бронирование**.
- Не используем в UI: `booking`, `booking flow`, `booking operations`.

### Парковка
- Основной термин: **парковка**.
- Для единицы размещения — **парковочное место**.
- Не используем в UI: «лот» без контекста.

### Статус
- Основной термин: **статус**.
- Для переходов: **действия по статусу** или **действия по жизненному циклу**.

### Неявка / no-show
- Основной термин: **незаезд** (в коротких элементах) или **не заехал** (статус).
- Не используем в UI: `no-show`.

### Жизненный цикл / lifecycle
- Основной термин: **жизненный цикл бронирования**.
- В коротких текстах: **действия по статусу**.
- Не используем в UI: `lifecycle`.

### Правила / rules
- Основной термин: **правила**.
- Не используем в UI: `rules`.

### Уведомления
- Основной термин: **уведомления**.

### Аналитика
- Основной термин: **аналитика**.
- `KPI` в заголовках заменяем на **ключевые показатели**.

### Управление
- Основной термин: **управление**.
- Подходит для административных и операционных разделов.

### Действия
- Основной термин: **действия**.
- Не выводим служебные коды действий (`open_details`, `change_status`) без перевода.

### Фильтры
- Основной термин: **фильтры**.

### Детали
- Основной термин: **детали**.

### Отмена / подтверждение / сохранение
- Кнопки: **Отменить**, **Подтвердить**, **Сохранить**.

## Рекомендации по длине UI copy
- Кнопки: 1–3 слова.
- Заголовки таблиц: до 2 слов, если смысл не теряется.
- Alert/подсказки: 1 короткое предложение, максимум 2.
