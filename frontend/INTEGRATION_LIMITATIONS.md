# Frontend integration limitations (backend gaps)

Список ограничений, обнаруженных во время проектирования frontend, на основе `docs/FRONTEND_READY_GAPS.md` и `docs/SCREEN_TO_API_MAPPING.md`.

## 1) Нет reference endpoint для enum/справочников
Frontend вынужден хранить роли/статусы/типы локально. Риск рассинхронизации при изменении backend enum.

## 2) Нет единого machine-readable error envelope
Ошибки в формате `detail` усложняют универсальные формы и toast-обработку.

## 3) Нет endpoint для capabilities
UI пока может показывать действия только на основе роли, а не ресурсно-зависимых разрешений.

## 4) Dashboard делает много отдельных запросов
Для экранов аналитики нужны несколько endpoint-ов и более сложный loading-state.

## 5) Нет realtime-канала
Уведомления/операционные изменения бронирований обновляются только polling-моделью.

## 6) Роль `uk` недостаточно документирована
Нужна дополнительная бизнес-спецификация, чтобы корректно определить UX и доступные маршруты.
